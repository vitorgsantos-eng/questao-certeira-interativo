## 2. VIT-27 — Script npm de testes

### Problema

`scripts/test-validation-logic.ts` existia, mas imprimia "PASSOU"/"FALHOU" em `console.log` sem rastrear falhas nem sair com código de erro. Uma falha nos testes não bloquearia a CI.

### Solução

Reescrita de `scripts/test-validation-logic.ts` seguindo o mesmo padrão dos scripts de sessão e rate limiting:
- Função `assert(condition, message)` com contadores `passed`/`failed`
- Saída `process.exit(1)` se qualquer teste falhar
- 3 Tiers: formato de código, hash/verify, lógica de expiração
- Não imprime hash completo, não usa dados reais

**Observação de bug encontrado:** o teste original incluía `QC-AB-123X` como código "inválido". O regex `^QC-[A-Z0-9]{2}-[A-Z0-9]{4}$` aceita letras no último segmento — `QC-AB-123X` é válido. Corrigido para `QC-AB-12345` (5 caracteres — genuinamente inválido).

### Script npm adicionado

Em `package.json`:

```json
"validate-content:ci": "tsx scripts/validate-content.ts content/revisions/revisao-9ano-triangulos-sistemas.json",
"test": "npx tsx scripts/test-validation-logic.ts && npx tsx scripts/test-session-signing.ts && npx tsx scripts/test-rate-limiting.ts"
```

- `validate-content:ci` — versão com arquivo fixo para uso em CI (sem argumento obrigatório)
- `test` — roda os 3 scripts de teste em sequência; qualquer falha aborta e retorna exit 1
