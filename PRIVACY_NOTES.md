# PRIVACY_NOTES.md — Privacidade e Dados

## Dados coletados no MVP

| Campo | Finalidade |
|-------|-----------|
| Apelido / primeiro nome | Personalizar a experiência |
| Série/turma | Associar ao nível de conteúdo |
| Código de acesso (hash) | Autenticação sem senha |
| Progresso por missão | Feedback de desempenho |
| Respostas (sim/não + alternativa escolhida) | Diagnóstico e relatório |
| Acertos e erros | Relatório pedagógico |
| Categoria de erro | Identificar padrões de dificuldade |
| Tempo aproximado | Estimativa de engajamento |
| Data de último acesso | Controle de expiração |

## Dados que NÃO são coletados

- CPF, RG, data de nascimento
- Endereço, localização
- Telefone, e-mail
- Foto ou imagem
- Dados de saúde ou vulnerabilidade
- Escola ou instituição (salvo se necessário para turma)
- Conversas livres ou respostas abertas
- Dados de redes sociais

## Princípios aplicados

**Minimização:** coletamos apenas o mínimo necessário para a finalidade pedagógica.

**Finalidade:** os dados são usados exclusivamente para fornecer feedback ao aluno e relatório ao professor. Não há uso comercial, publicidade ou compartilhamento.

**Acesso restrito:** apenas o professor/proprietário acessa o painel com dados dos alunos, via senha.

**Menores de idade:** o produto é destinado a menores. A coleta de dados é especialmente restrita. Nenhum dado sensível é coletado.

**Transparência:** o app informa o que é coletado ao acessar.

## Orientação para teste controlado

Durante o teste controlado:
- Informe os responsáveis sobre os dados coletados
- Use apelidos ou primeiros nomes, nunca nome completo se possível
- Não solicite foto ou informações além das listadas acima
- Ao encerrar o piloto, revise a necessidade de manter os dados

## Retenção de dados

Os dados são retidos enquanto o código de acesso está ativo.  
Após o piloto, o proprietário deve decidir sobre retenção ou exclusão dos dados.  
Scripts de exclusão podem ser criados sob demanda.
